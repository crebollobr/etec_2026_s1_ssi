## Autenticação, Autorização e Criptografia

Agora que a rede está protegida, precisamos garantir que as pessoas certas tenham acesso ao sistema e que os dados trafeguem em total sigilo.

### 📖 Teoria 

#### 1. Autenticação e Autorização no Linux
* **Autenticação (Quem é você?):** É o ato de provar sua identidade usando uma senha.
* **Autorização (O que você pode fazer?):** É a permissão. Você pode acessar o sistema, mas talvez não tenha autorização de deletar arquivos.
* No Linux, esses processos são gerenciados por duas engrenagens principais:
  * **NSS (Name Service Switch - `nsswitch.conf`):** É o "catálogo de endereços". Ele diz ao Linux onde procurar a lista de usuários válidos (pode ser em um arquivo local ou em um servidor na nuvem).
  * **PAM (Pluggable Authentication Modules):** É o "segurança da porta". Ele dita as regras de como a senha deve ser pedida, se precisa de letras maiúsculas, ou se o usuário deve ser bloqueado após errar a senha 3 vezes.

* **Hashing:** É a "impressão digital" de um arquivo. Ele transforma qualquer texto em um código fixo. Se o arquivo for alterado por um vírus, o Hash muda, e nós descobrimos a fraude.
* **Criptografia Simétrica:** Usa a *mesma chave* para trancar e destrancar a mensagem. É rápida, mas perigosa (como você envia a chave para o destinatário sem ninguém roubar?).
* **Criptografia Assimétrica (Chaves Pública e Privada):** Resolve o problema acima usando um par de chaves matemáticas:
  * **Chave Pública:** Você distribui para todo mundo. As pessoas usam ela para trancar (criptografar) mensagens para você.
  * **Chave Privada:** É o seu segredo. Só ela consegue destrancar (descriptografar) o que a chave pública trancou.
* **Certificados e Assinaturas Digitais:** Usam as chaves assimétricas para provar que um documento realmente foi enviado por você (assinatura) e que o site que você está acessando é autêntico (certificado SSL/cadeado verde).

### 💻 Laboratório Prático 02: O Mundo da Criptografia

Vamos investigar a autenticação do servidor e depois criar nossa própria comunicação secreta usando Chaves Assimétricas!

**Passo 1: Investigando o PAM e NSS**
1. Veja onde o Linux está procurando as senhas dos alunos:
   ```bash
   cat /etc/nsswitch.conf | grep passwd
   ```
2. Veja os módulos de segurança (PAM) do sistema:
   ```bash
   ls -l /etc/pam.d/
   ```

**Passo 2: Criptografia Assimétrica na Prática (OpenSSL)**
Neste laboratório, você vai criar um par de chaves, "trancar" uma mensagem com a chave pública e "destrancar" com a privada.

1. **Gere a sua Chave Privada (Nunca mostre a ninguém):**
   ```bash
   openssl genrsa -out chave_privada.pem 2048
   ```
2. **Extraia a sua Chave Pública (Para distribuir):**
   ```bash
   openssl rsa -in chave_privada.pem -pubout -out chave_publica.pem
   ```
3. **Crie uma mensagem secreta em texto claro:**
   ```bash
   echo "A senha do banco de dados e admin123" > segredo.txt
   ```
4. **Criptografe a mensagem (Usando a chave Pública):**
   ```bash
   openssl pkeyutl -encrypt -pubin -inkey chave_publica.pem -in segredo.txt -out segredo.enc
   ```
   *Dica: Tente ler o arquivo `segredo.enc` com o comando `cat segredo.enc`. Você verá apenas símbolos incompreensíveis!*
5. **Descriptografe a mensagem (Usando a chave Privada):**
   ```bash
   openssl pkeyutl -decrypt -inkey chave_privada.pem -in segredo.enc -out revelado.txt
   ```
6. **Leia o conteúdo final:**
   ```bash
   cat revelado.txt
   ```
