
# 🛡️ Curso Prático de Segurança da Informação

Bem-vindos ao nosso treinamento prático de Segurança da Informação! Neste curso de duas aulas, vamos explorar como proteger redes, identificar falhas e garantir a confidencialidade dos dados.

**Nossa infraestrutura:** Usaremos um servidor Linux compartilhado (1GB RAM). Para garantir que o servidor não trave, os alunos executarão os comandos essenciais, enquanto varreduras muito pesadas serão demonstradas apenas pelo instrutor.

---

## 📅 Aula 01: Defesas de Rede, Testes e Ferramentas Hacker

Nesta primeira aula, vamos entender as barreiras que protegem uma rede, como testamos a eficácia dessas barreiras e quais são as ferramentas de segurança mais famosas do mercado.

### 📖 Teoria (1 hora)

#### 1. Mecanismos de Segurança: Firewall, IDS e IPS
* **Firewall:** É a primeira linha de defesa. Ele analisa o tráfego que tenta entrar ou sair do servidor e aplica regras de bloqueio ou permissão. No Linux, a ferramenta nativa mais famosa para isso é o `iptables`.
* **IDS (Intrusion Detection System):** É o sistema de alarme. Ele monitora a rede em busca de comportamentos estranhos (como alguém tentando adivinhar senhas várias vezes) e avisa o administrador, mas não bloqueia a ação.
* **IPS (Intrusion Prevention System):** É o alarme conectado a um bloqueio automático. Ele detecta a invasão e corta a conexão do atacante na mesma hora.

#### 2. Testes de Penetração (Pentest) e Vulnerabilidades
Para saber se uma rede é segura, precisamos atacá-la! O *Pentest* é um ataque simulado e autorizado.
* **Vulnerabilidades:** São falhas no sistema. Lembram da nossa aula anterior sobre **SQL Injection**? Aquele é um exemplo clássico de vulnerabilidade em banco de dados que um pentest tenta descobrir antes que um criminoso o faça.

#### 3. Ferramentas Notáveis
Os profissionais de segurança (e os atacantes) usam um arsenal de ferramentas. As principais são:
* **Kali Linux:** Não é apenas uma ferramenta, mas um sistema operacional inteiro (uma versão do Linux) que já vem com centenas de programas de invasão pré-instalados.
* **NMAP:** O maior mapeador de redes. Ele descobre quais computadores estão ligados e quais portas estão abertas. *(Exemplo: Ele pode revelar que um servidor esqueceu a porta de banco de dados aberta para a internet).*
* **Wireshark:** Um "espião" de pacotes. Ele captura tudo o que passa pelo cabo de rede, permitindo ler conversas, e-mails ou senhas se não estiverem criptografados.
* **Metasploit:** Uma "fábrica de ataques". É um framework onde você escolhe uma vulnerabilidade (ex: uma falha do Windows antigo) e ele dispara o código para invadir a máquina.
* **Nessus:** Um scanner de vulnerabilidades corporativo. Ele varre a rede e gera um relatório dizendo: "O servidor X precisa ser atualizado, senão pode ser invadido".

### 💻 Laboratório Prático 01: Firewall e Escaneamento (1 hora)

Vamos acessar nosso servidor Linux e usar o `iptables` e o `nmap` na prática!

**Passo 1: Entendendo o Firewall (iptables)**
Vamos ver como o Linux bloqueia ou permite acessos.
1. Veja as regras atuais do firewall do nosso servidor:
   ```bash
   sudo iptables -L
   ```
2. Vamos criar uma regra para bloquear um IP suspeito (exemplo: `192.168.100.50`):
   ```bash
   sudo iptables -A INPUT -s 192.168.100.50 -j DROP
   ```
   *(A regra diz: Adicione na entrada (INPUT) que a origem (-s) 192.168.100.50 deve ser derrubada (DROP)).*

**Passo 2: O Ponto de Vista do Atacante (NMAP)**
Agora, vamos usar o NMAP para "atacar" nosso próprio servidor e ver o que o firewall está protegendo.
1. Faça um escaneamento básico no servidor local:
   ```bash
   nmap localhost
   ```
2. O instrutor rodará um escaneamento mais agressivo para mostrar a diferença (Apenas Instrutor):
   ```bash
   sudo nmap -sV -O localhost
   ```
   *(Esse comando tenta descobrir as versões dos programas rodando e o sistema operacional exato).*
