## Defesas de Rede, Testes e Ferramentas Hacker

Nesta primeira aula, vamos entender as barreiras que protegem uma rede, como testamos a eficácia dessas barreiras e quais são as ferramentas de segurança mais famosas do mercado.

### 📖 Teoria

#### 1. Mecanismos de Segurança: Firewall, IDS e IPS
* **Firewall:** É a primeira linha de defesa. Ele analisa o tráfego que tenta entrar ou sair do servidor e aplica regras de bloqueio ou permissão. No Linux, a ferramenta nativa mais famosa para isso é o `iptables`.
* **IDS (Intrusion Detection System):** É o sistema de alarme. Ele monitora a rede em busca de comportamentos estranhos (como alguém tentando adivinhar senhas várias vezes) e avisa o administrador, mas não bloqueia a ação.
* **IPS (Intrusion Prevention System):** É o alarme conectado a um bloqueio automático. Ele detecta a invasão e corta a conexão do atacante na mesma hora.

#### 2. Testes de Penetração (Pentest) e Vulnerabilidades
Para saber se uma rede é segura, precisamos atacá-la! O *Pentest* é um ataque simulado e autorizado.
* **Vulnerabilidades:** São falhas no sistema. Lembram da nossa aula anterior sobre **SQL Injection**? Aquele é um exemplo clássico de vulnerabilidade em banco de dados que um pentest tenta descobrir antes que um criminoso o faça.

#### 3. Ferramentas Notáveis
Os profissionais de segurança (e os atacantes) usam um arsenal de ferramentas. As principais são:
* **Kali Linux:** Não é apenas uma ferramenta, mas um sistema operacional inteiro (uma versão do Linux) que já vem com centenas de programas de invasão pré-instalados.
* **NMAP:** O maior mapeador de redes. Ele descobre quais computadores estão ligados e quais portas estão abertas. *(Exemplo: Ele pode revelar que um servidor esqueceu a porta de banco de dados aberta para a internet).*
* **Wireshark:** Um "espião" de pacotes. Ele captura tudo o que passa pelo cabo de rede, permitindo ler conversas, e-mails ou senhas se não estiverem criptografados.
* **Metasploit:** Uma "fábrica de ataques". É um framework onde você escolhe uma vulnerabilidade (ex: uma falha do Windows antigo) e ele dispara o código para invadir a máquina.
* **Nessus:** Um scanner de vulnerabilidades corporativo. Ele varre a rede e gera um relatório dizendo: "O servidor X precisa ser atualizado, senão pode ser invadido".

## 🗄️ Tabelas do iptables (Onde a mágica acontece)

No `iptables`, as **Tabelas** definem o *objetivo* da regra. Enquanto as *Chains* definem o momento em que o pacote é capturado, as *Tabelas* definem que tipo de processamento será feito nele (filtrar, traduzir endereço, alterar cabeçalho, etc).

O iptables possui 4 tabelas principais (e uma 5ª de segurança em sistemas mais modernos):

### 1. Tabela `filter` (A Padrão)
Se você não especificar uma tabela no comando (usando `-t`), o iptables usa a tabela `filter` por padrão. É aqui que decidimos se um pacote passa ou é bloqueado.
* **Função:** Filtragem de pacotes (Permitir/Bloquear).
* **Chains que utiliza:** `INPUT`, `FORWARD`, `OUTPUT`.

### 2. Tabela `nat` (Network Address Translation)

Usada exclusivamente para traduzir IPs ou Portas. É essa tabela que permite que uma rede inteira acesse a internet usando apenas um IP público (Roteador), ou que redireciona o tráfego de uma porta para outra.
* **Função:** Modificar IPs de origem ou destino (SNAT, DNAT, Masquerade).
* **Chains que utiliza:** * `PREROUTING` (Para DNAT / Redirecionamento de portas de entrada)
  * `POSTROUTING` (Para SNAT / Compartilhar internet)
  * `OUTPUT` (Para pacotes gerados localmente que precisam de NAT)
  * `INPUT` (Em kernels mais recentes, para NAT local).

### 3. Tabela `mangle`
Usada para "mastigar" ou alterar características avançadas e cabeçalhos dos pacotes TCP/IP antes do roteamento, como mudar o tempo de vida do pacote (TTL) ou marcar pacotes para controle de banda (QoS).
* **Função:** Alteração de pacotes e marcação.
* **Chains que utiliza:** Todas (`PREROUTING`, `INPUT`, `FORWARD`, `OUTPUT`, `POSTROUTING`).

### 4. Tabela `raw`
Usada antes de qualquer outra tabela para configurar exceções no rastreamento de conexões (Connection Tracking). Geralmente usada para otimizar o desempenho em servidores sob ataque ou com tráfego massivo.
* **Função:** Desativar o rastreamento de estado de pacotes específicos.
* **Chains que utiliza:** `PREROUTING`, `OUTPUT`.

### 🗺️ Resumo: Tabela x Chain

Para entender o fluxo completo, veja quais Chains estão disponíveis dentro de quais Tabelas:

| Tabela \ Chain | PREROUTING | INPUT | FORWARD | OUTPUT | POSTROUTING |
| :--- | :---: | :---: | :---: | :---: | :---: |
| **raw** | ✅ | ❌ | ❌ | ✅ | ❌ |
| **mangle** | ✅ | ✅ | ✅ | ✅ | ✅ |
| **nat** | ✅ | ✅* | ❌ | ✅ | ✅ |
| **filter** | ❌ | ✅ | ✅ | ✅ | ❌ |

*(A chain INPUT na tabela nat só está disponível no CentOS 7 / Ubuntu 14.04 ou superiores).*


Vamos acessar nosso servidor Linux e usar o `iptables` e o `nmap` na prática!

**Passo 1: Entendendo o Firewall (iptables)**
Vamos ver como o Linux bloqueia ou permite acessos.
1. Veja as regras atuais do firewall do nosso servidor:
   ```bash
   sudo iptables -L
   ```
2. Vamos criar uma regra para bloquear um IP suspeito (exemplo: `192.168.100.50`):
   ```bash
   sudo iptables -A INPUT -s 192.168.100.50 -j DROP
   ```
   *(A regra diz: Adicione na entrada (INPUT) que a origem (-s) 192.168.100.50 deve ser derrubada (DROP)).*
   * se for o destino podemos usar -d 192.168.100.50 *

**Passo 2: O Ponto de Vista do Atacante (NMAP)**
Agora, vamos usar o NMAP para "atacar" nosso próprio servidor e ver o que o firewall está protegendo.
1. Faça um escaneamento básico no servidor local:
   ```bash
   nmap localhost
   ```
2. O instrutor rodará um escaneamento mais agressivo para mostrar a diferença (Apenas Instrutor):
   ```bash
   sudo nmap -sV -O localhost
   ```
   *(Esse comando tenta descobrir as versões dos programas rodando e o sistema operacional exato).*


# 1. Limpa todas as regras e cadeias existentes
```
iptables -F
iptables -X
```

# 2. Define a Política Padrão como DROP (Bloqueia tudo por padrão)
```
iptables -P INPUT DROP
iptables -P FORWARD DROP
iptables -P OUTPUT DROP
```
# 3. Libera tráfego local (Loopback) - Essencial para o sistema
```
iptables -A INPUT -i lo -j ACCEPT
iptables -A OUTPUT -o lo -j ACCEPT
```
# 4. Libera pacotes de retorno (O estado "ESTABLISHED" permite a volta de conexões já iniciadas)
# Isso resolve o "pacote de volta" de forma global e segura.
```
iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
iptables -A OUTPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
```
# 5. Libera a entrada de Novas conexões para HTTP (80) e HTTPS (443)
```
iptables -A INPUT -p tcp --dport 80 -m state --state NEW -j ACCEPT
iptables -A INPUT -p tcp --dport 443 -m state --state NEW -j ACCEPT
```
# 6. Libera a saída de informação originada pelo PRÓPRIO servidor
# Permite que o servidor inicie novas conexões com a internet (para DNS, atualizações de pacotes, APIs, etc.)
```
iptables -A OUTPUT -m state --state NEW -j ACCEPT
```
