
# 🔐 Aula Prática: OAuth 2.0 — Login com Google

Na aula passada, cada aplicação guardava sua própria senha (em hash). Mas e se você pudesse **usar a sua conta do Google** para entrar em qualquer site, sem precisar criar uma senha nova? É isso que o **OAuth** faz.

## 1. O que é OAuth 2.0?

OAuth é o protocolo que permite que um site (a nossa aplicação Flask) peça ao Google para autenticar o usuário **sem nunca ver a senha dele**. É o famoso botão "Entrar com Google" / "Entrar com GitHub" / "Entrar com Facebook".

Os três personagens da história:

1. **O Usuário (Você):** quem está tentando entrar no site.
2. **A Aplicação (Flask):** nosso site, que quer saber quem é o usuário, mas não quer (e não deve) ver a senha dele.
3. **O Provedor de Identidade (Google):** quem conhece a senha do usuário e atesta a identidade dele para a Aplicação.

---

## 2. A "dança" do OAuth (Authorization Code Flow)

Passo a passo do que acontece quando você clica em **Entrar com Google**:

1. **Redirect para o Google:** A Aplicação manda o navegador do usuário para `accounts.google.com`, levando junto seu `client_id` (a identidade pública da Aplicação).
2. **Login no Google:** O usuário coloca usuário/senha **direto no Google**. A Aplicação nunca vê isso.
3. **Consentimento:** O Google pergunta ao usuário: *"Você autoriza essa Aplicação a ver seu e-mail, nome e foto?"*.
4. **Volta com um código:** O Google manda o navegador de volta para a Aplicação, passando um **código de autorização** temporário (`code`).
5. **Troca código por token:** A Aplicação chama o Google de novo, **agora servidor-para-servidor**, trocando o `code` + `client_secret` por um **access_token**.
6. **Pega os dados do usuário:** Com o `access_token`, a Aplicação chama uma API do Google e descobre o e-mail, nome e foto.
7. **Sessão criada:** Usuário logado!

> 💡 Repare que a senha do usuário **nunca passa pela nossa Aplicação**. Esse é o ponto principal do OAuth.

---

## 3. As chaves: `client_id` e `client_secret`

Para o Google saber que ele pode confiar na sua Aplicação, ela precisa ser **registrada** no Google Cloud Console. Em troca, o Google entrega duas coisas:

- **`client_id`:** identidade pública da Aplicação. Aparece na URL e pode ser visto pelo navegador do usuário.
- **`client_secret`:** "senha" da Aplicação. **Nunca pode ser exposta no navegador** — fica só no servidor.

> 🔑 **Nesta aula, o professor já registrou a Aplicação no Google e vai distribuir o `client_id` e o `client_secret` para todas as duplas.** Em produção, cada Aplicação tem o seu próprio par. No fim da aula a gente discute por quê.

## 4. Código pronto: Flask com Login do Google

Crie um arquivo chamado `app.py` com o código abaixo. As três variáveis no topo (`CLIENT_ID`, `CLIENT_SECRET`, `REDIRECT_URI`) são as únicas coisas que você precisa preencher.

```python
from flask import Flask, request, redirect, session
from urllib.parse import urlencode
import requests
import secrets

app = Flask(__name__)
# Chave usada para assinar o cookie de sessão. Em produção, use algo aleatório e secreto.
app.secret_key = "troque-isso-por-uma-string-aleatoria"

# --- CHAVES DO GOOGLE (recebidas do professor) ---
CLIENT_ID = "COLE_AQUI_O_CLIENT_ID"
CLIENT_SECRET = "COLE_AQUI_O_CLIENT_SECRET"

# Troque "timeXX" pelo nome da SUA dupla (time01, time02, ...)
REDIRECT_URI = "https://curso.chr.eti.br/timeXX/callback"

# --- Endpoints públicos do Google ---
AUTH_URL = "https://accounts.google.com/o/oauth2/v2/auth"
TOKEN_URL = "https://oauth2.googleapis.com/token"
USERINFO_URL = "https://www.googleapis.com/oauth2/v2/userinfo"


@app.route("/")
def home():
    # Se o usuário já fez login, mostra os dados dele
    if "user" in session:
        u = session["user"]
        return f"""
            <h1>Olá, {u['name']}!</h1>
            <p>E-mail: {u['email']}</p>
            <img src="{u['picture']}" alt="foto" />
            <p><a href="/logout">Sair</a></p>
        """
    # Caso contrário, mostra o botão de login
    return '<h1>Bem-vindo!</h1><a href="/login">Entrar com Google</a>'


@app.route("/login")
def login():
    # Passo 1: gerar um "state" aleatório para proteger contra CSRF
    state = secrets.token_urlsafe(16)
    session["oauth_state"] = state

    # Passo 2: montar a URL de autorização do Google e redirecionar o navegador
    params = {
        "client_id": CLIENT_ID,
        "redirect_uri": REDIRECT_URI,
        "response_type": "code",
        "scope": "openid email profile",
        "state": state,
    }
    return redirect(AUTH_URL + "?" + urlencode(params))


@app.route("/callback")
def callback():
    # Passo 3: o Google nos devolveu o usuário com um "code" na URL
    code = request.args.get("code")
    state = request.args.get("state")

    # Verificação de segurança: o state precisa bater com o que mandamos
    if state != session.get("oauth_state"):
        return "Erro: state inválido (possível ataque CSRF)", 400

    # Passo 4: trocar o code por um access_token (servidor -> servidor)
    token_response = requests.post(TOKEN_URL, data={
        "client_id": CLIENT_ID,
        "client_secret": CLIENT_SECRET,
        "code": code,
        "grant_type": "authorization_code",
        "redirect_uri": REDIRECT_URI,
    }).json()

    access_token = token_response["access_token"]

    # Passo 5: usar o access_token para pegar os dados do usuário no Google
    userinfo = requests.get(
        USERINFO_URL,
        headers={"Authorization": f"Bearer {access_token}"},
    ).json()

    # Passo 6: salvar o usuário na sessão (cookie assinado pelo Flask)
    session["user"] = {
        "name": userinfo["name"],
        "email": userinfo["email"],
        "picture": userinfo["picture"],
    }
    return redirect("/")


@app.route("/logout")
def logout():
    session.clear()
    return redirect("/")


if __name__ == "__main__":
    app.run(host="0.0.0.0", port=5000, debug=True)
```
