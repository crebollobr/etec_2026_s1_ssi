# 🚀 Lab: Teste de Vulnerabilidade com OWASP ZAP (Sem Instalação)

**Cenário:** O Desktop tem 4GB de RAM e o sistema está travado. Vamos rodar tudo via arquivos portáteis (ZIP) no diretório de Downloads.

---

## 1. Preparação do Ambiente
Crie a pasta base para organizar a bagunça:
1. Abra o **Explorador de Arquivos**.
2. Vá em `Downloads`.
3. Crie uma pasta chamada **web**.

### Downloads Necessários (Arquivos .zip):
* **Java (JRE 17+ Portable):** Extrair dentro de `Downloads`. (Renomeie a pasta extraída para `java` para facilitar).
* **OWASP ZAP (Cross-platform):** Extrair dentro de `Downloads`.
* **app.py:** Salvar dentro da pasta `Downloads\web`. 
    > **Dica:** Abra o [link do GitHub](https://github.com/crebollobr/etec_2026_s1_ssi/blob/main/app.py), clique em **"Raw"**, dê um `Ctrl+S` e salve como `app.py` na pasta `web`.

---

## 2. Terminal 1: O Alvo (Servidor Python)
Abra o primeiro CMD e execute os comandos para criar o ambiente isolado do Python:

```cmd
:: Entrar na pasta do projeto
cd %USERPROFILE%\Downloads\web

:: Criar e ativar o ambiente virtual (venv)
python -m venv venv
.\venv\Scripts\activate.bat

:: Instalar as dependências (sqlite3 já vem no Python)
pip install flask

:: Rodar o site vulnerável
python app.py
```
*Mantenha esta tela aberta! O servidor estará rodando em `http://localhost:5000`.*



---

## 3. Terminal 2: O Atacante (OWASP ZAP)
Abra um **novo** CMD. Agora vamos chamar o ZAP usando o Java que você baixou (sem usar o Java do sistema que está desatualizado).

```cmd
:: Entrar na pasta onde o ZAP foi extraído (ajuste o nome da pasta se necessário)
cd %USERPROFILE%\Downloads\ZAP_2.17.0_Crossplatform\ZAP_2.17.0

:: Rodar o ZAP usando o caminho relativo para o seu Java portátil
:: O comando abaixo assume que a pasta 'java' está um nível acima
..\..\java\bin\java.exe -Xmx2G -jar zap-2.17.0.jar
```

---

## 4. O Ataque (Mão na Massa)

### Passo A: Conhecendo o inimigo
1. Abra o navegador do Windows e acesse `http://localhost:5000`.
2. Tente fazer um login manual com qualquer usuário para ver o site funcionando.

### Passo B: O Spider (Mapeamento)
1. No **ZAP**, clique na aba **Início Rápido**.
2. Clique em **Varredura Automatizada**.
3. Em "URL para ataque", digite: `http://localhost:5000`.
4. Clique em **Ataque**.
5. Observe na aba **Sites** (esquerda) que o ZAP "enxergou" o formulário de login.

### Passo C: Varredura Ativa (Exploração)
1. Na aba **Sites**, clique com o botão direito em `http://localhost:5000`.
2. Vá em **Ataque -> Varredura Ativa**.
3. Aguarde a barra de progresso. Quando terminar, vá na aba **Alertas**.



---

## 5. Checkpoint de Aprendizagem
O ZAP deve encontrar uma **Bandeira Vermelha (SQL Injection)**. 
* Peça para o aluno clicar no alerta e ler a **"Requisição"**. 
* Ele verá que o ZAP tentou injetar algo como `admin' OR '1'='1' --`.

---

### ⚠️ Notas Importantes para o Lab:
* **Caminhos:** Se o aluno extraiu o Java com outro nome (ex: `jdk-17.0.1`), ele precisará ajustar o comando do Terminal 2.
* **Memória:** O parâmetro `-Xmx2G` é fundamental. Ele garante que o Java não tente usar toda a RAM do PC, evitando que o Windows trave.
* **Limpeza:** Ao final da aula, basta deletar a pasta `web` e os zips em `Downloads`. Nada fica instalado no registro do Windows.

Essa estrutura de dois CMDs separados ajuda o aluno a entender a diferença entre o **Servidor (Vítima)** e a **Ferramenta de Auditoria (Atacante)**.
