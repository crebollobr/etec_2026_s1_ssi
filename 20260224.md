# Familia ISO/IEC 27000

## 1. ISO/IEC 27000: O Dicionário

**O que é:** Define a terminologia padrão. Parece simples, mas em segurança, todos precisam falar a mesma língua para evitar erros.

* **Exemplo:** Ela define exatamente o que é um "Ativo", uma "Ameaça" e uma "Vulnerabilidade".
* **Na prática:** Se um auditor fala sobre "Não conformidade", ele e o gestor estão usando a definição exata contida na 27000 para que não haja confusão jurídica ou técnica.

## 2. ISO/IEC 27001: O Manual de Requisitos

**O que é:** É a "estrela" da família. É a única que estabelece **requisitos obrigatórios** para que uma empresa seja certificada. Ela foca no modelo de gestão (SGSI).

* **Exemplo:** Ela exige que a alta direção esteja comprometida e que os riscos sejam tratados.
* **Na prática:** Uma empresa de software quer provar para um banco que é segura. Ela implementa a 27001, passa por auditoria e recebe o selo oficial.

## 3. ISO/IEC 27002: O Catálogo de Controles

**O que é:** Enquanto a 27001 diz "você deve ter controles", a 27002 explica **como** esses controles devem ser. É um guia de boas práticas detalhado.

* **Exemplo:** A 27001 diz: "proteja as senhas". A 27002 detalha: "as senhas devem ter X caracteres, ser trocadas periodicamente e não serem anotadas em papel".
* **Na prática:** O TI usa a 27002 para configurar o firewall e as políticas de acesso no dia a dia.

## 4. ISO/IEC 27003: O Guia de Implementação

**O que é:** É o passo a passo para quem está começando do zero. Ela oferece orientações sobre como projetar o SGSI antes de colocá-lo em prática.

* **Exemplo:** Ajuda a definir o "escopo" (quais departamentos da empresa entrarão na certificação).
* **Na prática:** Um consultor usa a 27003 para criar o cronograma de implementação do projeto de segurança em uma fábrica.

## 5. ISO/IEC 27004: O Painel de Métricas

**O que é:** Focada em medição. Como saber se a segurança está funcionando? "O que não se mede, não se gerencia".

* **Exemplo:** Define como calcular a eficácia dos treinamentos de segurança.
* **Na prática:** O gestor apresenta um gráfico para o CEO mostrando que, após a implementação dos controles, o número de incidentes de vírus caiu **80%** (isso é aplicação da 27004).

## 6. ISO/IEC 27005: A Gestão de Riscos

**O que é:** Foca exclusivamente em identificar, analisar e tratar riscos. É considerada uma das partes mais críticas, pois a segurança deve ser baseada no risco.

* **Exemplo:** Ajuda a decidir: "É mais perigoso um hacker invadir o site ou um funcionário perder um notebook?".
* **Na prática:** A empresa identifica que o risco de inundação no CPD é alto. Com base na 27005, decide mover os servidores para o 2º andar ou para a nuvem.

## 7. ISO/IEC 27006: A Regra para os Auditores

**O que é:** Esta norma não é para as empresas comuns, mas para as **certificadoras** (como a BSI ou TÜV). Ela garante que os auditores sejam competentes e imparciais.

* **Exemplo:** Define que um auditor precisa de X horas de experiência para poder certificar alguém.
* **Na prática:** Garante que o selo ISO 27001 que você vê na parede de uma empresa tenha valor real, pois quem o deu seguiu as regras rígidas da 27006.

### Resumo Comparativo

| Norma | Papel Principal | Público-alvo |
| --- | --- | --- |
| **27000** | Vocabulário | Todos os envolvidos. |
| **27001** | **Certificação** (O "O Quê") | A Organização como um todo. |
| **27002** | Prática (O "Como") | Equipes de TI e Segurança. |
| **27003** | Planejamento | Gestores de projeto. |
| **27004** | Medição | Analistas e Gestores de KPI. |
| **27005** | Riscos | Analistas de Risco. |
| **27006** | Auditoria | Empresas que emitem certificados. |
