## **Aula de Segurança Web (2 Horas)**

### **1. Mozilla Web Security & HTTPS**
As diretrizes da Mozilla focam em configurar o servidor para que o navegador se torne uma barreira. 
* **HTTPS/TLS:** Protege os dados em trânsito. Sem isso, um hacker na mesma rede Wi-Fi lê tudo. 
* **HSTS:** Um cabeçalho que diz ao navegador: "Nunca mais me acesse via HTTP, apenas HTTPS".



---

### **2. OWASP Top 10 e Proactive Controls**
A OWASP lista os maiores riscos. Dois dos principais são:
1.  **Injeção (XSS):** O atacante insere um script malicioso no seu site.
2.  **Configuração Incorreta:** Esquecer os headers de segurança que veremos agora.

### **4. Experimento 1: O Site Inseguro (Vulnerável)**

Crie o arquivo `app_vulneravel.py`. Ele simula um site que confia em tudo o que o usuário digita e não impõe limites ao navegador.

```python
from flask import Flask, request

app = Flask(__name__)

@app.route('/')
def index():
    # O problema: Pegamos o que vem da URL e jogamos direto no HTML
    nome = request.args.get('nome', 'Visitante')
    
    return f"""
    <html>
        <body>
            <h1>Bem-vindo, {nome}</h1>
            <p>Este site esta vulneravel a XSS e falta de CORS.</p>
            <div id="dados-sensiveis">Saldo do Usuario: R$ 5.000,00</div>
        </body>
    </html>
    """

# Rota de API que qualquer site pode ler (falta de CORS)
@app.route('/api/privado')
def api():
    return {"cartao": "4444-5555-6666-7777", "cvv": "123"}

if __name__ == '__main__':
    app.run(port=5000)
```

**Como testar a falha:**
1.  **Ataque XSS:** Acesse `http://localhost:5000/?nome=<script>alert('Hacked!')</script>`. O navegador executará o alerta porque não há **CSP**.
2.  **Falha de CORS:** Abra o site `google.com`, aperte **F12**, vá no **Console** e cole:
    `fetch('http://localhost:5000/api/privado').then(r => r.json()).then(console.log)`
    O navegador pode permitir a leitura desses dados sensíveis por um site estranho.

### **5. Experimento 2: O Site Seguro (Mozilla & OWASP)**

Crie o arquivo `app_seguro.py`. Aqui aplicamos as defesas detalhadamente.

```python
from flask import Flask, request, make_response
import html

app = Flask(__name__)

@app.route('/')
def index():
    nome = request.args.get('nome', 'Visitante')
    # SOLUÇÃO OWASP: Sanitizar a entrada (limpar tags HTML)
    nome_seguro = html.escape(nome)
    
    conteudo = f"<h1>Bem-vindo, {nome_seguro}</h1><p>Site protegido com CSP e CORS.</p>"
    res = make_response(conteudo)

    # --- SOLUÇÃO MOZILLA: Headers de Segurança ---
    
    # CSP: Diz ao navegador para SÓ rodar scripts que venham do MEU próprio servidor
    res.headers['Content-Security-Policy'] = "default-src 'self'"
    
    # X-Frame-Options: Impede que seu site seja "sequestrado" dentro de um iframe em outro site
    res.headers['X-Frame-Options'] = 'DENY'
    
    return res

@app.route('/api/privado')
def api():
    res = make_response({"status": "Acesso negado por CORS"})
    # CORS: Definimos que NENHUM site externo pode ler esta API
    res.headers['Access-Control-Allow-Origin'] = 'none'
    return res

if __name__ == '__main__':
    app.run(port=5001)
```

### **6. Verificação dos Resultados**

| Teste | Resultado no Site Seguro (Porta 5001) | Explicação Técnica |
| :--- | :--- | :--- |
| **Injeção de Script** | O script aparece como texto puro, não executa. | O `html.escape` neutralizou as tags `<script>`. |
| **Teste de CSP** | No Console (F12) aparece um erro em vermelho. | O Header **CSP** bloqueou a execução de qualquer código não autorizado. |
| **Teste de CORS** | O `fetch` feito de outro site é bloqueado. | O navegador viu que o site de origem não está na lista permitida pelo servidor. |



### **Resumo dos Conceitos Explicados:**
* **CORS:** É o porteiro que decide se o site A pode ler dados do site B.
* **CSP:** É a lista branca que diz ao navegador quais scripts e imagens são "confiáveis".
* **XSS:** É o ataque de injetar código onde não deveria existir código.
* **Mozilla Guidelines:** É o manual de como configurar esses "porteiros" e "listas" corretamente.
