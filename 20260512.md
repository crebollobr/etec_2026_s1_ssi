
# 🔐 Aula Prática: Autenticação Web e Segurança de Senhas

Nesta aula, vamos entender como a internet lida com o seu login e como construímos um sistema de autenticação seguro. Vamos usar a **Autenticação HTTP Básica**, que é aquela janela nativa de login que o próprio navegador (Chrome, Firefox) abre na tela.

## 1. O Básico: Como funciona a Autenticação na Web?

Quando você entra em um site restrito, o processo de login acontece nestes 4 passos:

1. **O Login:** O usuário digita o seu login e sua senha (neste laboratório, na janela do próprio navegador).
2. **A Criptografia (Hash):** O servidor recebe a senha em texto puro e a passa por uma função matemática complexa, transformando-a em um **Hash** (um código longo e embaralhado).
3. **A Comparação:** O sistema nunca compara a sua senha com outra senha em texto. Ele vai no Banco de Dados, busca o Hash que está salvo lá, e compara se o Hash gerado na hora é igual ao Hash armazenado. 
4. **A Sessão (Cookie / Headers):** Estando tudo OK, o site progride. Na web tradicional, o servidor envia um "Cookie" para manter a sessão. Na Autenticação HTTP Básica, o próprio navegador assume esse papel, guardando a senha na memória e enviando-a automaticamente (via *headers*) a cada nova página que você acessa, para você não precisar digitar de novo.



---

## 2. O que são os Hashes?

Um algoritmo de Hash transforma qualquer texto em uma "impressão digital" de tamanho fixo. A regra de ouro da segurança web é: **é impossível pegar um Hash e transformá-lo de volta na senha original.**

Existem vários tipos de hashes:
* **MD5:** Antigo, gera um código de 32 caracteres.
* **SHA-1:** Antigo, gera um código de 40 caracteres.
* **SHA-256:** Padrão moderno e seguro, gera um código de 64 caracteres.

## 3. Exemplo Prático: Site com verificação em MD5 (Inseguro)

Abaixo, temos o código de um site real em Python (usando o framework Flask). Ele usa a janela de autenticação nativa do navegador e verifica o login usando o **MD5**.

### Por que o MD5 não é seguro e teve que ser abandonado?
O MD5 se tornou rápido demais para os computadores modernos. Hoje, um computador comum consegue testar bilhões de combinações de senhas por segundo. Hackers usam tabelas gigantes (Rainbow Tables) ou força bruta para quebrar um hash MD5 em minutos. Por isso, ele nunca mais deve ser usado para guardar senhas.

### Preparando o ambiente
Abra o seu terminal na pasta do projeto e crie seu ambiente virtual:

**No Windows:**
```bash
python -m venv venv
venv\Scripts\activate
pip install flask

```

**No Linux / Mac:**

```bash
python3 -m venv venv
source venv/bin/activate
pip install flask

```

Crie um arquivo chamado `app.py` e cole o código abaixo:

```python
from flask import Flask, request, Response
import hashlib

app = Flask(__name__)

# --- NOSSO "BANCO DE DADOS" FALSO ---
# O usuário é "aluno". 
# O Hash abaixo é a versão MD5 da senha "secreto123"
USUARIOS = {
    "aluno": "6d4173db2a351fbc5563e4a318a5d247"
}

def verifica_credenciais(username, password):
    # ATENÇÃO: Convertendo a senha digitada em MD5
    hash_senha_digitada = hashlib.md5(password.encode()).hexdigest()
    
    # Compara se o usuário existe e se o hash bate
    return username in USUARIOS and USUARIOS[username] == hash_senha_digitada

@app.route('/')
def home():
    # Pega os dados da janela nativa do navegador
    auth = request.authorization

    # Se o usuário não digitou nada ou errou a senha, abre a janela do navegador
    if not auth or not verifica_credenciais(auth.username, auth.password):
        return Response(
            'Acesso Negado! Você precisa fazer login.', 
            401, 
            {'WWW-Authenticate': 'Basic realm="Login do Sistema"'}
        )
    
    # Se a senha estiver certa (o hash bater), o site progride
    return f"<h1>Acesso Liberado!</h1> <p>Bem-vindo, {auth.username}. Você está logado no sistema.</p>"

if __name__ == '__main__':
    app.run(debug=True)

```

Rode o comando `python app.py` no terminal, abra o navegador em `http://127.0.0.1:5000` e veja a janela de Autenticação HTTP aparecer. Teste logar com **aluno** e **secreto123**.

## 4. Laboratório do Aluno: Migrando para SHA-256

O sistema acima funciona, mas está vulnerável porque usa MD5. **O seu trabalho agora é corrigir o código do servidor `app.py` para usar SHA-256.**

### O que você precisa fazer:

**Passo 1: Gerar o novo Hash**
Abra o interpretador do Python no seu terminal e gere o hash SHA-256 da senha `secreto123` (ou de uma nova senha da sua escolha):

```python
import hashlib
print(hashlib.sha256("secreto123".encode()).hexdigest())

```

**Passo 2: Atualizar o Banco de Dados**
No arquivo `app.py`, substitua o hash MD5 antigo pelo novo hash SHA-256 gigante que você acabou de gerar dentro do dicionário `USUARIOS`.

**Passo 3: Atualizar a Função de Verificação**
Vá até a função `verifica_credenciais` no `app.py`. Mude a linha que usa `hashlib.md5(...)` para usar a função de segurança correta.

**Passo 4: Testar**
Reinicie o servidor (`python app.py`) e tente logar novamente. Se você conseguir entrar, parabéns! Você acabou de fechar uma falha de segurança grave em uma aplicação web.
