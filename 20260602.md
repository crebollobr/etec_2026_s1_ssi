## 📚 Parte 1: Fundamentos Teóricos da Arquitetura

Antes de rodar o código, precisamos entender por que grandes empresas (como Google, Microsoft e bancos) separam os seus sistemas em dois sites diferentes.

### 1. Separação de Responsabilidades (Delegação)

Na arquitetura moderna, as aplicações que o usuário usa (o Site Cliente) **nunca devem encostar na senha do usuário**.

* **O Provedor de Identidade (Site 1 na porta 8001):** É o cofre da empresa. É o único site que tem o banco de dados de usuários, senhas e que exige o Segundo Fator (2FA).
* **O Site Cliente (Site 2 na porta 8002):** É a aplicação web. Ele é "cego" para credenciais. Se ele precisa saber quem é o usuário, ele delega essa tarefa para o Site 1.

### 2. O Protocolo OAuth 2.0 e OpenID Connect (OIDC)

Para que o Site 2 confie no Site 1 sem expor a senha do usuário, eles conversam usando um padrão global de mercado através de uma "dança" de mensagens.

Essa comunicação acontece em dois canais diferentes:

* **Front-channel (Via Navegador do Usuário):** O Site Cliente redireciona o navegador para o Servidor de Autenticação. O usuário digita a senha e o 2FA. O Servidor de Autenticação devolve o navegador para o Site Cliente trazendo um **Código de Autorização** (`code`) na URL. Esse código é temporário, de uso único, e se for roubado, não serve para nada sozinho.
* **Back-channel (Servidor conversando com Servidor):** O Site Cliente pega esse código da URL e, por trás dos panos (sem o usuário ver), faz um `POST` direto para o Servidor de Autenticação. Ele diz: *"Aqui está o código do usuário e aqui está a minha senha de aplicativo (`client_secret`)"*. Se tudo bater, o Servidor de Autenticação entrega o **Access Token** (Token de Acesso), que é a chave real para liberar o sistema.

### 3. O Segundo Fator (2FA)

O 2FA (Two-Factor Authentication) garante que, mesmo que um hacker descubra a senha do banco de dados, ele não consiga logar. A autenticação exige algo que o usuário *sabe* (a senha) e algo que o usuário *tem* (um código gerado no celular dele). Toda essa validação pesada fica isolada no Servidor de Autenticação.

## 💻 Parte 2: O Código do Servidor de Autenticação (`servidor_auth.py`)

Crie uma pasta, entre nela e crie o arquivo `servidor_auth.py`.
Este site simula o provedor de identidade (rodando na porta **8001**). Ele recebe o usuário, valida a senha e o Segundo Fator (2FA), e emite o token de acesso.

```python
from flask import Flask, request, redirect, render_template_string, jsonify
import secrets

app = Flask(__name__)

# --- BANCO DE DADOS DO SERVIDOR DE AUTENTICAÇÃO ---
CLIENTES_CADASTRADOS = {
    "meu-site-cliente": "senha-secreta-do-site"
}

USUARIOS_CADASTRADOS = {
    "aluno": {"senha": "123", "2fa": "9999", "email": "aluno@curso.chr.eti.br"}
}

# Memória temporária para chaves e códigos do protocolo
codigos_autorizacao = {}
tokens_acesso = {}

@app.route("/auth")
def auth():
    client_id = request.args.get("client_id")
    redirect_uri = request.args.get("redirect_uri")
    state = request.args.get("state")
    
    if client_id not in CLIENTES_CADASTRADOS:
        return "Erro: Site Cliente não homologado neste servidor.", 403

    # Tela de Login que exige a Senha + Código de 2º Fator
    return render_template_string("""
        <h2>Servidor Central de Autenticação (:8001)</h2>
        <form action="/login" method="POST">
            <input type="hidden" name="redirect_uri" value="{{ redirect_uri }}">
            <input type="hidden" name="state" value="{{ state }}">
            <p>Usuário: <input type="text" name="username" required></p>
            <p>Senha: <input type="password" name="password" required></p>
            <p>Código 2FA: <input type="text" name="token_2fa" required></p>
            <button type="submit">Autenticar e Validar 2FA</button>
        </form>
    """, redirect_uri=redirect_uri, state=state)

@app.route("/login", methods=["POST"])
def login():
    username = request.form.get("username")
    password = request.form.get("password")
    token_2fa = request.form.get("token_2fa")
    redirect_uri = request.form.get("redirect_uri")
    state = request.form.get("state")

    user = USUARIOS_CADASTRADOS.get(username)
    
    # Valida credenciais + Segundo Fator
    if user and user["senha"] == password and user["2fa"] == token_2fa:
        codigo_temp = secrets.token_hex(16)
        codigos_autorizacao[codigo_temp] = username
        # Devolve o navegador para o domínio do Site Cliente com o código
        return redirect(f"{redirect_uri}?code={codigo_temp}&state={state}")
    
    return "Credenciais ou código 2FA inválidos!", 401

@app.route("/token", methods=["POST"])
def token():
    # Validação interna (Site Cliente conversando com o Servidor de Autenticação)
    client_id = request.form.get("client_id")
    client_secret = request.form.get("client_secret")
    code = request.form.get("code")

    if CLIENTES_CADASTRADOS.get(client_id) != client_secret:
        return jsonify({"erro": "Client Secret inválido"}), 401

    if code in codigos_autorizacao:
        username = codigos_autorizacao.pop(code)
        access_token = secrets.token_hex(32)
        tokens_acesso[access_token] = username
        return jsonify({"access_token": access_token})
    
    return jsonify({"erro": "Código de autorização inválido ou expirado"}), 400

@app.route("/userinfo")
def userinfo():
    auth_header = request.headers.get("Authorization")
    if not auth_header or not auth_header.startswith("Bearer "):
        return jsonify({"erro": "Token ausente"}), 401
    
    token = auth_header.split(" ")[1]
    if token in tokens_acesso:
        username = tokens_acesso[token]
        email = USUARIOS_CADASTRADOS[username]["email"]
        return jsonify({"preferred_username": username, "email": email})

    return jsonify({"erro": "Token inválido"}), 401

if __name__ == "__main__":
    app.run(host="0.0.0.0", port=8001, debug=True)

```

## 💻 Parte 3: O Código do Site Cliente (`servidor_cliente.py`)

Na mesma pasta, crie o arquivo `servidor_cliente.py`.
Este site (rodando na porta **8002**) não gerencia senhas. Ele aponta todas as URLs diretamente para o domínio correto do Servidor de Autenticação.

```python
from flask import Flask, request, redirect, session
from urllib.parse import urlencode
import requests
import secrets

app = Flask(__name__)
app.secret_key = "chave-super-secreta-da-sessao"

# --- CONFIGURAÇÃO DE PORTAS DA REDE ---
PORTA_AUTH = "8001"
PORTA_CLIENTE = "8002"

# --- CONFIGURAÇÃO DE CREDENCIAIS ---
CLIENT_ID = "meu-site-cliente"
CLIENT_SECRET = "senha-secreta-do-site"

# --- ENDPOINTS APONTANDO PARA O DOMÍNIO CORRETO DO SERVIDOR ---
AUTH_URL = f"http://curso.chr.eti.br:{PORTA_AUTH}/auth"
TOKEN_URL = f"http://curso.chr.eti.br:{PORTA_AUTH}/token"
USERINFO_URL = f"http://curso.chr.eti.br:{PORTA_AUTH}/userinfo"


@app.route("/")
def home():
    if "user" in session:
        u = session["user"]
        return f"""
            <h1>Acessado com Sucesso! Site Cliente (: {PORTA_CLIENTE})</h1>
            <p><strong>Usuário Logado via Provedor:</strong> {u['preferred_username']}</p>
            <p><strong>E-mail institucional:</strong> {u['email']}</p>
            <p><a href="/logout">Sair do Sistema</a></p>
        """
    return f'<h1>Site Cliente Privado (:{PORTA_CLIENTE})</h1><a href="/login">Entrar via Servidor de 2FA</a>'


@app.route("/login")
def login():
    state = secrets.token_urlsafe(16)
    session["oauth_state"] = state

    # MENSAGEM 1: Redireciona o navegador do aluno usando o domínio público da porta 8001
    params = {
        "client_id": CLIENT_ID,
        "redirect_uri": f"http://curso.chr.eti.br:{PORTA_CLIENTE}/callback",
        "state": state,
    }
    return redirect(AUTH_URL + "?" + urlencode(params))


@app.route("/callback")
def callback():
    code = request.args.get("code")
    state = request.args.get("state")

    if state != session.get("oauth_state"):
        return "Erro de Segurança: O parâmetro state está inválido ou corrompido!", 400

    # MENSAGEM 2: Requisição via backchannel enviada ao domínio do Servidor de Autenticação
    dados_post = {
        "client_id": CLIENT_ID,
        "client_secret": CLIENT_SECRET,
        "code": code,
    }
    
    resposta_token = requests.post(TOKEN_URL, data=dados_post).json()
    access_token = resposta_token.get("access_token")

    if not access_token:
        return "Erro: O Servidor de Autenticação recusou a emissão do token.", 400

    # MENSAGEM 3: Busca informações do perfil do usuário utilizando o token recebido
    cabecalhos = {"Authorization": f"Bearer {access_token}"}
    userinfo = requests.get(USERINFO_URL, headers=cabecalhos).json()

    # Registra a sessão local no Site Cliente
    session["user"] = {
        "preferred_username": userinfo.get("preferred_username"),
        "email": userinfo.get("email"),
    }
    return redirect("/")


@app.route("/logout")
def logout():
    session.clear()
    return redirect("/")


if __name__ == "__main__":
    app.run(host="0.0.0.0", port=8002, debug=True)

```
